### 교차신호 아키텍처 (v5.1 갱신)

축간 연동으로 단일 축에서 보이지 않는 **전신 위기 징후**를 포착한다.
**순환축(축1~8)끼리만 교차 검사** — 구조축(축9)은 제외 [교정E].

| Phase | 교차 조합 | 패턴 수 | 상태 |
|-------|-----------|---------|------|
| Phase 1 (현재) | 축1↔축5, 축1↔축7, 축1↔축8, 축5↔축7, 축7↔축8, **축4↔축7** | **6** | ✅ 구현 완료 |
| Phase 2 (예정) | 축2/3/6 연동 | 미정 | 📋 데이터 확보 후 |

**교차부스트 로직 [교정C]:**
- min(la,lb) ≥ 2 AND severity ≥ 4 → +1.0 (전이 확인)
- severity ≥ 3 → +0.5 (전이 가능성)
- 기타 → 0

**축 4 팩트체크 정의 (v1.2 신규)**

```
4B 공정률 팩트체크
  = 건설 진행 신호: SAR 측정 건물 형태 변화(높이·면적)가
    공식 공정률 보고와 동일 방향인가
  판정: 공정률 +20%p 보고인데 SAR 높이 변화 0m → 괴리 플래그

4C 실입주 팩트체크
  = 활동·사용 신호: 야간광량 + 주차장 점유 + 난방열이
    공식 입주율과 동일 수준인가
  판정: 입주율 90% 보고인데 야간광량이 인근 대비 30% 미만 → 괴리 플래그

4D 괴리 경보
  = 공식 통계와 위성 신호의 방향 불일치가 2개월 이상 지속될 때 발동
  원칙: 단발 괴리는 노이즈, 지속 괴리는 조작 또는 구조 문제
```
